1、开发内容
	
	淘宝首页（移动端）

2、开发时间

	1.5天

3、遇到的问题及解决方案

	html 自带滚动条的处理

	    解决：overflow: hidden;

	使用 rem 单位时繁琐的计算

	    解决：在设置字号时取合适的值，如100px;提高开发效率；


4、项目描述

	移动端项目，在不同分辨率下内容可以还原显示；

	还原显示处理：计算像素比后设置视口；

	布局内容结构化，使用与内容相关的命名，便于二次开发与修改；

	使用 html5 新增标签，遵从标签语义化，关爱浏览器，优化SEO；

	使用了CSS3样式，优化页面交互体验；如：使用 animation 与 transform 等实现 banner 图轮播；

	使用了新属性 placeholder 处理 input 中的预留文字；

	使用 rem 单位设置数值，还原在不同分辨率下的表现；


5、该项目的技术总结

	移动端项目中，目的是网页在不同设备中表现相同；

	设置视口是为了在不同设备下，所设置的数值能够完整还原；
		
	    各个设备的DPR值不统一，因此要首先获取各设备的像素比，依据像素比计算并设置相应的视口；

	另一方面，一个盒子以像素为单位，在大小不同分辨率的屏幕上虽然以其正确尺寸显示，但是表现有差别；
	
	    使用document.documentElement.clientWidth获取当前设备的屏幕分辨率，并再次基础上设置 html 字号大小
	
	    以 rem 为单位，基于html字号大小去设置每个元素的大小，那么每个元素的大小都是基于当前设备去计算大小的；

	在 banner 图轮播中为了看起来连贯，在html中增加的相同的图片标签；

	项目中对于同级元素中有不同的样式，css3选择器的应用更加方便，不需要在对不同元素设置单独的class，优化代码；


	不足：开发过程还不够熟练，需要在更多项目中实践。
